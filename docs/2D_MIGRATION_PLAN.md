# 2Dドットライク画面への移行計画

## 📋 概要

3D実装の複雑さによる問題を解決するため、2Dドットライクな画面に全面作り直します。

## 🎯 設計方針

### 1. 画面構成
- **上部**: 地上エリア（明るい青空色）
- **中央**: 地面（茶色の帯）
- **下部**: 地下エリア（暗い色、深度が深いほど暗く）
- **背景**: 深度に応じたグラデーション（シンプルなCSSグラデーション）

### 2. 感情オブジェクトの表示
- 2Dドット/ピクセルアート風のアイコン
- 感情カテゴリごとに異なる形状と色
- 深度に応じた位置に配置（スクロールで移動）

### 3. 植物システム
- 2Dスプライト風のアニメーション
- 感情オブジェクトの周囲に配置
- シンプルなCSSアニメーションで動きを表現

### 4. スクロール機能
- 既存の`useDepth`フックを維持
- 背景色とオブジェクトの位置を深度に応じて更新

## 🛠 実装計画

### Phase 1: 基本構造の作成
1. `Scene2D.tsx`コンポーネントの作成
2. 背景色のグラデーション実装
3. 基本的なレイアウト

### Phase 2: 感情オブジェクトの表示
1. 2D感情オブジェクトコンポーネントの作成
2. 深度に応じたフィルタリング
3. クリック/ホバーインタラクション

### Phase 3: 植物システム
1. 2D植物コンポーネントの作成
2. 感情オブジェクトとの連携
3. アニメーション

### Phase 4: 統合
1. `DepthCanvas.tsx`を2D版に切り替え
2. 既存のUIコンポーネントとの統合
3. テストと調整

## 📝 技術スタック

- **React**: 既存のコンポーネント構造を維持
- **CSS/Tailwind**: スタイリングとアニメーション
- **Framer Motion**: アニメーション（必要に応じて）
- **Canvas API**: ドットライクな描画（オプション）

## ✅ メリット

1. **シンプル**: 3Dの複雑さを排除
2. **パフォーマンス**: 軽量で高速
3. **デバッグ**: 問題の特定が容易
4. **保守性**: コードが理解しやすい

## ⚠️ 注意点

1. 既存の3Dアセットは使用不可
2. 没入感は3Dより劣る可能性
3. 実装時間が必要
